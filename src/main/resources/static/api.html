Overview

VulScan is an internal vulnerability management backend that correlates asset inventory data with public vulnerability sources. The API is consumed by a lightweight dashboard UI and inventory clients.

  • KEV data is fetched from the CISA Known Exploited Vulnerabilities feed
  • Inventory is imported as CSV from client scripts
  • Risk views combine inventory and KEV information

Dashboard

GET /api/dashboard/summary

Returns aggregated dashboard statistics.

{
  "totalHosts": 12,
  "totalInventoryItems": 404,
  "kevEntries": 1234,
  "lastKevRefresh": "2026-02-05"
}

GET /api/dashboard/hosts

Returns per-host inventory summary.

[
  {
    "hostname": "PC-001",
    "softwareCount": 42
  }
]

KEV Catalog

GET /api/kev/status

Returns last KEV refresh date and result.

{
  "lastRefreshDate": "2026-02-05",
  "lastResult": "OK (1234 entries stored)"
}

POST /api/kev/refresh

Triggers a refresh from the CISA KEV feed.

{
  "inserted": 1234,
  "updated": 0,
  "message": "OK"
}

GET /api/kev/latest

Returns latest KEV entries for dashboard tables.

[
  {
    "cveId": "CVE-2026-0001",
    "vendor": "ExampleVendor",
    "product": "ExampleProduct",
    "dateAdded": "2026-02-01"
  }
]

GET /api/kev/catalog

Returns the full KEV catalog (use carefully).

Inventory

POST /api/inventory/import

Imports inventory items from CSV (multipart/form-data).

{
  "rowsRead": 404,
  "rowsInserted": 404,
  "rowsRejected": 0,
  "errors": []
}

CSV format

hostname,product,vendor,version,installedOn,source,cpe

GET /api/inventory

Lists inventory items.

Risk

GET /api/risk/top

Returns highest-risk assets based on KEV correlation.

[
  {
    "hostname": "PC-001",
    "riskScore": 0.92,
    "kevCount": 5
  }
]

Error handling

Errors follow standard HTTP status codes.

{
  "status": 400,
  "error": "Bad Request",
  "message": "Missing CSV file"
}

cURL examples

curl http://localhost:8080/api/kev/status
curl -X POST http://localhost:8080/api/kev/refresh
curl -F "file=@C:\SoftwareInventory.csv" http://localhost:8080/api/inventory/import
